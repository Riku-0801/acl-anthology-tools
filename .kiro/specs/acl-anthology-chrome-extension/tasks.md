# Implementation Plan

- [x] 1. プロジェクト基盤の構築

- [x] 1.1 Chrome Extension MV3の設定ファイル群を作成する
  - `manifest.json`にMV3形式でコンテンツスクリプト・Service Worker・Popup・`host_permissions`（`https://aclanthology.org/*`）を定義する
  - TypeScript strict modeとVite + vite-plugin-web-extensionのビルド設定を構成する
  - `chrome.storage.local`と`clipboardWrite`のパーミッションを設定する
  - URLパターン別にコンテンツスクリプトをイベントページ用・一覧ページ用・詳細ページ用の3つにマッピングする
  - _Requirements: 1.1, 2.1, 3.1, 4.1, 5.1, 6.1, 7.1_

- [x] 1.2 ACL AnthologyのDOMから論文情報を抽出するユーティリティを実装する
  - 一覧ページの論文エントリ（タイトル・著者・PDFリンク・BibTeXリンク・アブストラクト）を抽出する関数を実装する
  - 詳細ページから単一論文エントリ（タイトル・著者・アブストラクト・BibTeX全文）を抽出する関数を実装する
  - すべてのセレクター文字列を定数ファイルに集約して変更箇所を局所化する
  - セレクターで要素が見つからない場合は`null`を返すフォールバック処理を実装する
  - _Requirements: 1.1, 2.1, 3.1, 5.1_

- [x] 2. データ永続化サービスの実装

- [x] 2.1 (P) ブックマーク・既読記録・設定の読み書きサービスを実装する
  - `chrome.storage.local`への型付きアクセスを提供するStorageServiceを実装する
  - ブックマーク追加・削除・一覧取得・存在確認の操作を実装する
  - 既読マーク・訪問済み自動記録・記録取得の操作を実装する
  - アブストラクトプレビューの有効/無効など拡張機能設定の読み書きを実装する
  - ストレージ使用量が9MBを超えた場合に警告を返す容量チェックを実装する
  - _Requirements: 4.1, 4.2, 4.4, 7.1, 7.2, 7.3_

- [x] 2.2 (P) ブックマークと既読記録のエクスポート機能を実装する
  - ブックマーク一覧をJSON形式でシリアライズしてダウンロードする機能を実装する
  - 既読・訪問済み記録をCSV形式（paperId, isRead, markedReadAt, visitedAt）でダウンロードする機能を実装する
  - _Requirements: 4.5, 7.5_

- [x] 3. ネットワーク・フェッチサービスの実装

- [x] 3.1 (P) Service WorkerのBibTeX・著者情報フェッチハンドラーを実装する
  - `chrome.runtime.onMessage`でコンテンツスクリプトからのメッセージを受け取る
  - `FETCH_BIBTEX`メッセージで指定URLにfetchして結果を返す
  - `FETCH_AUTHOR_PAPERS`メッセージで著者ページをfetchしてパースした論文リストを返す
  - フェッチ失敗時は`{ ok: false, error: 'fetch_failed' }`形式でエラーを返す
  - _Requirements: 1.1, 1.3, 6.4_

- [x] 3.2 (P) BibTeX取得・クリップボードコピーサービスを実装する
  - 詳細ページDOMの`<div id="citeBibtex"><pre>`からBibTeXテキストを直接取得する関数を実装する
  - 一覧ページでは`.bib` URLをService Worker経由でフェッチしてBibTeXを取得する関数を実装する
  - `navigator.clipboard.writeText`でクリップボードにコピーし、失敗時はフォールバックUIを提供する
  - _Requirements: 1.1, 1.2, 1.3, 1.4_

- [x] 3.3 (P) 著者論文フェッチとキャッシュ管理サービスを実装する
  - 著者スラッグをキーに`/people/{slug}/`ページをService Worker経由でフェッチして最新論文リストを取得する
  - 取得結果を`chrome.storage.local`の`authorCache`キーに保存し、60分のTTLで再フェッチを制御する
  - キャッシュヒット時はフェッチをスキップしてキャッシュデータを返す
  - _Requirements: 6.1, 6.5_

- [x] 4. 検索サービスとイベントページ検索機能の実装

- [x] 4.1 (P) 論文リストのテキストフィルタリングサービスを実装する
  - クエリ文字列と論文エントリ配列を受け取り、マッチするエントリのインデックスセットを返す純粋関数を実装する
  - タイトル・著者名・アブストラクトの各フィールドを大文字小文字区別なし部分一致で検索する
  - アブストラクトが未定義のエントリはタイトルと著者名のみで評価する
  - クエリが空文字の場合は全件のインデックスセットを返す
  - _Requirements: 5.2, 5.3, 5.6_

- [x] 4.2 イベントページに検索バーを注入してリアルタイム絞り込みを実装する
  - `aclanthology.org/events/`配下のページでDOMContentLoaded後に論文エントリを全件収集してインデックスを構築する
  - 論文一覧の上部に検索バーHTML要素を挿入する
  - `input`イベントごとにSearchServiceを呼び出し、非マッチエントリに`hidden`属性を付与して非表示にする
  - 検索バー付近にマッチ件数を「N件 / 全M件」形式で表示するカウンターを更新する
  - 検索バーを空にすると全エントリの`hidden`属性を除去して全件再表示する
  - _Requirements: 5.1, 5.2, 5.4, 5.5, 5.6_

- [x] 5. 論文一覧ページのBibTeX・PDFアクセス機能の実装

- [x] 5.1 一覧ページの各論文エントリにBibTeXコピーボタンを注入する
  - ListingContentScriptで論文エントリのホバー時にBibTeXコピーボタンを表示するUIを実装する
  - ボタンクリック時にBibtexServiceでBibTeXを取得してクリップボードにコピーする
  - コピー成功時は「コピーしました」のフィードバックを一時表示し、失敗時はエラーメッセージと論文ページへのリンクをインライン表示する
  - _Requirements: 1.1, 1.2, 1.3_

- [x] 5.2 一覧ページの各論文エントリにPDF直接アクセスボタンを注入する
  - DomExtractorで取得したPDFリンクURLをボタンとして各エントリに表示する
  - ボタンクリックでPDFを新しいタブで開く
  - ArXivバージョンのリンクが存在する場合は追加オプションとして表示する
  - PDFリンクが存在しない場合はボタンをグレーアウトして代替リンクを表示する
  - _Requirements: 3.1, 3.2, 3.3, 3.4_

- [x] 6. アブストラクトプレビュー機能の実装

- [x] 6.1 論文タイトルのホバー時にアブストラクトをポップアップ表示する
  - ListingContentScriptで論文タイトルへのホバーイベントを検出してポップアップUIを表示する
  - ポップアップ内にアブストラクトテキストとBibTeXコピー・PDF開く・ブックマーク追加のアクションボタンを配置する
  - Escapeキー押下でポップアップを閉じる
  - アブストラクトプレビューの有効/無効をStorageServiceの設定から読み込んで制御する
  - _Requirements: 2.1, 2.2, 2.3, 2.5_

- [x] 6.2 ページ内にアブストラクトが存在しない場合の非同期取得を実装する
  - DomExtractorがアブストラクトを取得できなかったエントリに対して、論文詳細ページをService Worker経由でフェッチする
  - フェッチ中はポップアップ内にスケルトンUIを表示する
  - フェッチ完了後にアブストラクトテキストをポップアップに描画する
  - _Requirements: 2.4_

- [x] 7. ブックマーク機能の実装

- [x] 7.1 一覧ページの各論文エントリにブックマークボタンを実装する
  - 各論文エントリのホバー時にブックマークボタンを表示する
  - ボタンクリック時にStorageServiceを通じて論文をブックマーク保存し、ボタンを「保存済み」状態に更新する
  - すでにブックマーク済みの論文を再クリックした場合は確認の上ブックマークを解除する
  - ページロード時にStorageServiceから既存のブックマーク状態を読み込んでボタン表示を同期する
  - _Requirements: 4.1, 4.4_

- [x] 7.2 PopupのブックマークリストUIを実装する
  - 拡張機能アイコンクリック時に開くPopup HTMLにStorageServiceから保存済み論文一覧を表示する
  - 各ブックマークエントリにタイトル・著者・保存日時を表示し、クリックで該当論文ページを新しいタブで開く
  - _Requirements: 4.2, 4.3_

- [x] 8. 既読・未読トラッキング機能の実装

- [x] 8.1 一覧ページの各論文エントリに既読マークボタンと視覚的状態表示を実装する
  - 各論文エントリに既読マークボタンを追加し、クリックでStorageServiceに既読状態を保存する
  - ページロード時にStorageServiceから既読状態を読み込み、既読エントリにチェックマークや色変更などの視覚的インジケーターを表示する
  - 訪問済み（`visitedAt`記録済み）のエントリは既読マークとは別の視覚スタイルで区別して表示する
  - _Requirements: 7.1, 7.2_

- [x] 8.2 既読・未読フィルタリングUIを実装する
  - 一覧ページに「すべて表示」「未読のみ」「既読のみ」を切り替えるフィルターコントロールを実装する
  - フィルター切り替え時にStorageServiceから既読状態を取得してエントリの表示・非表示を制御する
  - _Requirements: 7.4_

- [x] 9. 著者探索機能の実装

- [x] 9.1 著者名ホバー時に最近の論文ポップアップを表示する
  - 一覧ページの著者名リンクへのホバーイベントを検出してAuthorServiceで著者データを取得する
  - ポップアップに著者の最近の論文（最大5件）のタイトルと「全著作を見る」リンクを表示する
  - 著者データ取得失敗時は「情報を取得できませんでした」と著者ページへの直接リンクを表示する
  - 同一著者のポップアップが短時間に再表示される場合はキャッシュを利用してフェッチをスキップする
  - _Requirements: 6.1, 6.2, 6.4, 6.5_

- [x] 10. 論文詳細ページ機能の統合

- [x] 10.1 詳細ページにBibTeXコピー・ブックマーク・訪問記録自動保存を実装する
  - DetailContentScriptでページロード時に論文IDをACL Anthology IDパターンで検証し処理を開始する
  - ページ上部にワンクリックBibTeXコピーボタンを常時表示し、`<div id="citeBibtex"><pre>`から直接BibTeXを取得してコピーする
  - ブックマークボタンをページに配置し、StorageServiceとの状態同期を実装する
  - ページ初回訪問時にStorageServiceの`markAsVisited`を自動呼び出して訪問記録を保存する
  - _Requirements: 1.4, 4.1, 4.4, 7.3_

- [x] 10.2 詳細ページに著者サイドパネルを実装する
  - 論文詳細ページの共著者リストからAuthorServiceで各著者の代表論文を取得してサイドパネルに表示する
  - 著者情報取得失敗時は「情報を取得できませんでした」と著者ページへの直接リンクを表示する
  - _Requirements: 6.3, 6.4, 6.5_

- [x] 11. Popupのエクスポート機能を実装する
  - Popup UIにブックマークをJSONエクスポートするボタンと既読記録をCSVエクスポートするボタンを追加する
  - ボタンクリック時にStorageServiceのエクスポート関数を呼び出してブラウザダウンロードをトリガーする
  - _Requirements: 4.5, 7.5_

- [x] 12. テストの実装

- [x] 12.1 コアサービスのユニットテストを実装する
  - SearchServiceのフィルタリングロジック（空クエリ・部分一致・大小文字・アブストラクト未定義フォールバック）をテストする
  - DomExtractorがACL AnthologyのHTMLスナップショットから正しく論文情報を抽出できることをテストする
  - StorageServiceのブックマーク追加・削除・既読マーク・エクスポートの状態遷移をテストする
  - BibtexServiceのDOM取得・クリップボードコピーをモック環境でテストする
  - _Requirements: 1.2, 1.3, 4.1, 4.4, 5.2, 5.3, 5.6, 7.1_

- [x] 12.2 コンテンツスクリプトの統合テストを実装する
  - EventsContentScriptが検索バー注入・入力・DOM絞り込み・件数表示・クリアを正しく動作させることを確認する
  - ListingContentScriptのBibTeXコピーボタンがService Workerへのメッセージ送信とフィードバック表示を正しく行うことを確認する
  - ブックマークボタンクリックでchrome.storageへの書き込みが発生することを確認する
  - _Requirements: 1.1, 1.2, 1.3, 4.1, 4.2, 5.1_

- [ ]* 12.3 主要ユーザーフローのE2Eテストを実装する
  - イベントページでの検索フロー（テキスト入力 → 絞り込み → 件数表示 → クリアで全件表示）をPlaywrightでテストする
  - 一覧ページでのBibTeXコピーフロー（ボタンクリック → クリップボード確認 → フィードバック表示）をテストする
  - ブックマーク管理フロー（追加 → Popup確認 → 削除確認）をテストする
  - _Requirements: 1.1, 1.2, 4.1, 4.2, 4.3, 4.4, 5.1, 5.4, 5.5_
